List of main PyKEP elements¶

Name Type Description
PyKEP.epoch class represents an epoch (i.e. a fixed point in time)

PyKEP.epoch_from_iso_string() function helper function to construct an epoch from a string containing a date in the ISO format YYYYMMDDTHHMMSS
PyKEP.planet class represents a generic planet in a keplerian motion around its primary

PyKEP.planet_gtoc5 class represents an asteroid from the GTOC5 competition asteroid list
PyKEP.planet_gtoc2 class represents an asteroid from the GTOC2 competition asteroid list
PyKEP.lambert_problem class solves the multirevolution lambert problem
PyKEP.propagate_lagrangian() function propagates pure keplerian motion using Lagrange coefficients and universal variables
PyKEP.propagate_taylor() function propagates keplerian motion disturbed by a constant inertial thrust using Taylor integration method

Detailed Documentation¶

class PyKEP.lambert_problem(*args)¶

Represents a multiple revolution Lambert’s problem

__init__(*args)¶

lambert_problem(r1, r2, t [, mu = 1, cw = False])

  • r1: starting position (x1,y1,z1)
  • r2: 3D final position (x2,y2,z2)
  • t: time of flight
  • mu: gravitational parameter
  • cw: True for clock-wise motion, False otherwise

NOTE: Units need to be consistent.

NOTE: The multirev Lambert’s problem will be solved upon construction and its solution stored in data members.

Example (non-dimensional units used):

l = lambert_problem([1,0,0],[0,1,0],5 * pi / 2. )
get_v1()¶

Returns a sequence of vectors containing the velocities at r1 of all computed solutions to the Lambert’s Problem

Solutions are stored in order 0 rev, 1rev, 1rev, 2rev, 2rev, ...

Example (extracts v1 for the 0 revs solution):

v10 = l.get_v1()[0]
get_v2()¶

Returns a sequence of vectors containing the velocities at r2 of all computed solutions to the Lambert’s Problem

Solutions are stored in order 0 rev, 1rev, 1rev, 2rev, 2rev, ...

Example (extracts v2 for the 0 revs solution):

v20 = l.get_v2()[0]
get_a()¶

Returns a sequence containing the semi-major axes of all computed solutions to the Lambert’s Problem

Solutions are stored in order 0 rev, 1rev, 1rev, 2rev, 2rev, ...

Example (extracts a for the 0 revs solution):

a0 = l.get_a()[0]
get_p()¶

Returns a sequence containing the semilatus rectum (parameter) of all computed solutions to the Lambert’s Problem

Solutions are stored in order 0 rev, 1rev, 1rev, 2rev, 2rev, ...

Example (extracts p for the 0 revs solution):

p0 = l.get_p()[0]
get_Nmax()¶

Returns the maximum number of revolutions for which a solution exists. The total number of solution to the Lambert’s problem will thus be n_sol = Nmax*2 + 1

Example:

Nmax = l.get_Nmax()
n_sol = Nmax*2+1
get_iters()¶

Returns a sequence containing the number of iterations employed to compute each solution to the Lambert’s Problem

Solutions are stored in order 0 rev, 1rev, 1rev, 2rev, 2rev, ...

Example (extracts the number of iterations employed for the 0 revs solution):

p0 = l.get_iters()[0]
is_reliable()¶

Returns True if all solutions to the Lambert’s problem have been solved using less iterations than the maximum number of iterations allowed. This is set at compile time in the c++ header file astro_constants.h

Example:

if l.is_reliable():
  n_sol = len(l.get_a())
class PyKEP.epoch(*args)¶

Represents a precise point in time. The boost::posix_time_ptime classes are used to handle the conversion to and from string

__init__(*args)¶

PyKEP.epoch(jd[, jd_type=epoch.epoch_type.MJD2000])

  • jd: a julian date
  • jd_type: julian date type, see PyKEP.epoch.epoch_type

Examples:

e1 = epoch(0)
e1 = epoch(0,epoch.epoch_type.MJD2000)
e2 = epoch(54333, epoch.epoch_type.MJD)
jd()¶

jd = e.jd()
mjd()¶

Returns the Modified Julian Date

mjd2000()¶

Returns the Modifeid Julian Date 2000

  • PyKEP.epoch.epoch_type.MJD2000
PyKEP.epoch_from_string(s)¶

PyKEP.epoch_from_string(s)

  • s: string containing a date in the format ‘YYYY-MM-DD HH:MM:SS’Returns a PyKEP.epoch object constructed from a from a delimited string containing a date.Excess digits in fractional seconds will be dropped. Ex: ‘1:02:03.123456999’ => ‘1:02:03.123456’.This behavior depends on the precision defined in astro_constant.h used to compile.

NOTE: The function is based on the corresponding boost date_time library function

Example:

e = PyKEP.epoch_from_string('2002-01-20 23:59:54.003')
PyKEP.epoch_from_iso_string(s)¶

PyKEP.epoch_from_iso_string(s)

  • s: string containing a date in the iso format ‘YYYYMMDDTHHMMSS’Returns a PyKEP.epoch object constructed from a from a non delimited iso form string containing a date.

NOTE: The function is based on the corresponding boost date_time library function

Example:

e = PyKEP.epoch_from_iso_string('20020120T235954')
class PyKEP.planet(*args)¶

A planet ... contains the ephemerides calculations

__init__(*args)¶

PyKEP.planet(when,orbital_elements, mu_central_body, mu_self,radius, safe_radius [, name = ‘unknown’])

  • when: a PyKEP.epoch indicating the orbital elements epoch
  • orbital_elements: a sequence of six containing a,e,i,W,w,M (SI units, i.e. meters and radiants)
  • mu_central_body: gravity parameter of the central body (SI units, i.e. m^2/s^3)
  • mu_self: gravity parameter of the planet (SI units, i.e. m^2/s^3)
  • radius: body radius (SI units, i.e. meters)
  • safe_radius: body distance safe for a spacecraft fly-by
  • name: body name

NOTE: use the derived classes PyKEP.planet_ss, PyKEP.planet_mpcorb to instantiate common objectsExample:

earth = planet(epoch(54000,epoch.epoch_type.MJD),(9.9998805e-01 * AU, 1.6716812e-02, 8.8543531e-04 * DEG2RAD, 1.7540648e+02 * DEG2RAD, 2.8761578e+02 * DEG2RAD, 2.5760684e+02 * DEG2RAD), MU_SUN, 398600.4418e9, 6378000, 6900000,  'Earth'
eph(*args)¶

PyKEP.planet.eph(when)

  • when: a PyKEP.epoch indicating the epoch at which the ephemerides are needed

Retuns a tuple containing the planet position and velocity in SI units

Example:

r,v = earth.eph(epoch(5433))
orbital_elements(*args)¶

PyKEP.planet.orbital_elements(when)

  • when: a PyKEP.epoch indicating the epoch at which the orbital elements are needed

Returns a tuple containing the six orbital elements a,e,i,W,w,M at the desired epoch (SI units used)

Example:

elem = earth.orbital_elements(epoch(5433))

PyKEP.planet.orbital_elements()

Returns a tuple containing the six orbital elements a,e,i,W,w,M at the reference epoch (SI units used)

Example:

elem = earth.orbital_elements()

class PyKEP.planet_gtoc2(*args)¶
__init__(*args)¶

PyKEP.planet_gtoc2(ast_id)

  • ast_id: Construct from a consecutive id from 0 to 910 (Earth).The order is that of the original data file from JPL

    • Group 1: 0 - 95
    • Group 2: 96 - 271
    • Group 3: 272 - 571
    • Group 4: 572 - 909
    • Earth: 910

earth_gtoc2 = planet_gtoc2(910)
PyKEP.propagate_lagrangian(*args)¶

PyKEP.propagate_lagrangian(r,v,t,mu)

  • r: start position, x,y,z
  • v: start velocity, vx,vy,vz
  • t: propagation time
  • mu: central body gravity constant

Returns a tuple containing r and v, the final position and velocity after the propagation.

Example:

r,v = propagate_lagrangian([1,0,0],[0,1,0],pi/2,1)
PyKEP.propagate_taylor(*args)¶

PyKEP.propagate_taylor(r,v,m,u,t,mu,veff,log10tol,log10rtol)

  • r: start position, x,y,z
  • v: start velocity, vx,vy,vz
  • m: starting mass- t: propagation time
  • u: fixed inertial thrust, ux,uy,uz
  • mu: central body gravity constant
  • veff: the product (Isp g0)
  • log10tol: the logartihm of the absolute tolerance passed to Jorba’s‘ taylor propagator
  • log10rtol: the logartihm of the relative tolerance passed to Jorba’s‘ taylor propagator

Returns a tuple containing r, v, and m the final position, velocity and mass after the propagation.

Example:

r,v,m = propagate_taylor([1,0,0],[0,1,0],100,[0,0,0],pi/2,1,1,-15,-15)
